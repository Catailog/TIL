# 2025-12-23 TIL

## 오늘 배운 것

- OSI 7계층, 헤더, 쿠키, 세션
- 과거에는 비표준/실험용 헤더 키와 MIME 타입에 `x-` 접두사 사용했었으나 현대 표준에서는 새로운 헤더/타입을 정의할 때 `x-` 접두사 사용 권장하지 않는다. 표준화 과정에서 혼동이 발생하기 때문이다.
  - 이름을 변경하지 않는다면 비표준인지, 표준인지 이름만으로 알 수 없음
  - 이름을 변경한다면 호환성에 문제가 생김
- 쿠키에 저장된 세션 ID와 만료 정보 등을 통해 서버에서는 해당 세션을 식별하고 유지할 수 있다. HTTP는 무상태(stateless) 프로토콜이기 때문에 사용자를 식별하기 위해 세션이 필요하다. 클라이언트 측에서 쿠키에 정보들을 저장하는 이유는 제어, 보안 측면에서 유리하기 때문이다. 사실상 쿠키가 유일한 선택지이며, 세션에 사용할 수 있는 선택지들의 특징은 아래와 같다.
  - URL
    - 로그, 히스토리, 북마크 등 정보가 남아 위험
  - on-memory(HTML / JS 변수)
    - 새로고침 시 데이터를 잃음
  - LocalStorage / SessionStorage
    - HTTP 요청에 자동 포함 안 됨
    - XSS에 취약
  - 쿠키
    - HTTP 요청마다 자동으로 전송됨
    - 보안 옵션 제공
