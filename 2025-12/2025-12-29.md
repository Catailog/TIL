# 2025-12-29 TIL

## 오늘 배운 것

- 크로스 브라우징, jQuery, SASS/SCSS, 디버깅, 렌더링
- 과거에는 표준이 없었기 때문에 브라우저마다 코드를 다르게 작성해야 했고, 서로 다른 브라우저에서 같은 동작을 지원하기 위해 브라우저에 따라 기능을 제공하는 것을 크로스 브라우징이라 한다. 이러한 크로스 브라우징을 잘 지원해주는 라이브러리가 바로 jQuery이다.
- jQueyr를 모던 개발환경에서는 거의 사용하지 않는다. 레거시 프로젝트를 유지보수해야 되는 경우를 제외하면 볼 일이 극히 드물다. 다음 2가지 이유 때문에 사용되지 않는다.
  - js 번들 사이즈 증가 부담(일부 코드만 사용하더라도 전체를 로드해야 함)
  - 불필요한 연산 처리 부담(현대 브라우저에서는 표준을 따르기 때문에 브라우저별 분기가 불필요)
- 브라우저 렌더링 과정
  - HTML 파싱 후 DOM 생성
  - CSS 파싱 후 CSSOM 생성
  - 이때 script를 만나면 blocking 되고, 이를 방지하려면 defer/async가 있어야 함
  - DOM과 CSSOM을 합쳐져 Render Tree 생성
  - Layout(각 요소의 위치, 크기 계산)
  - Paint(픽셀 단위로 화면에 그리기)
- 스크립트에 의해 Layout/Paint를 다시 계산해 렌더링 하는 것을 reflow/repaint라 한다. reflow에 의해 다른 요소들의 위치, 크기에 영향을 미칠 수 있기 때문에 잦은 reflow 발생 시 성능이 저하될 수 있어 주의가 필요하다.
- CSS보다 간결하게 작성하기 위해 2006년 SASS가 탄생했다. 처음엔 Python 스타일의 들여쓰기 기반 문법을 채택했으나 2010년에 CSS와 100% 호환되도록 중괄호와 세미콜론을 허용하는 SCSS가 새로 나왔다.
- SCSS의 특징
  - 중첩(Nesting, 관련된 요소들을 구조화)
  - 변수(Variables) - CSS에도 도입됨
  - 믹스인/함수(Mixin, 코드 재사용)
  - 연산 - CSS에도 도입됨
  - 상속(Inheritance)
  - 모듈화(파일을 여럿으로 분할하여 import)
