## 오늘 배운 것

- API Mocking, a11y

- msw(Mock Service Worker)는 브라우저의 Service Worker를 활용해 네트워크 수준에서 API 요청을 가로채고 모의 응답을 반환하는 API Mocking 라이브러리이다. 실제 백엔드로부터의 응답을 받는 것처럼 에러, 로딩에 대한 Mocking 테스트를 편리하게 할 수 있다.
- 웹 접근성(Web Accessibility)은 모든 사용자가 정보에 평등하게 접근할 수 있도록 보장하는 네 가지 핵심 원칙 POUR를 기반으로 설계된다.
  - Perceivable (인식 가능성): 정보와 인터페이스 구성 요소는 사용자의 감각(시각, 청각 등)으로 인지될 수 있어야 한다. 핵심 요소: 이미지의 대체 텍스트(alt) 제공, 자막 제공, 텍스트와 배경의 충분한 명도 대비 확보.
  - Operable (운용 가능성): 사용자가 다양한 입력 장치를 통해 인터페이스를 제어하고 탐색할 수 있어야 한다. 핵심 요소: 모든 기능을 키보드만으로 조작 가능, 충분한 응답 시간 제공, 광과민성 발작을 유발하는 디자인 회피.
  - Understandable (이해 가능성): 콘텐츠의 내용과 인터페이스의 작동 방식이 논리적이고 예측 가능해야 한다. 핵심 요소: 기본 언어 설정(lang 속성), 일관된 네비게이션 구조, 입력 오류 시 명확한 수정 안내.
  - Robust (견고성): 기술의 변화에 관계없이 보조 기술(스크린 리더 등)을 포함한 다양한 환경에서 안정적으로 해석되어야 한다. 핵심 요소: 웹 표준 문법 준수, 시맨틱 마크업(Semantic Markup) 활용, 최신 브라우저 및 보조 기기와의 호환성 유지.
