## 오늘 배운 것

- Firebase Authentication/Firestore Database

- Firebase에서 가져온 키 등의 정보를 담은 .env 파일로부터 config 객체를 생성하여 initializeApp(config)을 통해 Firebase SDK를 초기화할 수 있다. 그리고 getAuth(app), getFirestore(app)으로 Firebase Authentication 인스턴스와 Cloud Firestore 인스턴스를 생성할 수 있다.
- getFirestore를 통해 얻은 db에 다음과 같은 방식으로 접근할 수 있다.

```ts
getDocs(query(collection(db, "collectionName"), limit(1)));
```

- Firestore은 NoSQL이기 때문에 데이터 한 행을 record가 아니라 document라고 칭한다. 마찬가지로 table이라는 용어 대신 collection이라는 용어가 쓰인다. getDocs가 실질적으로 쿼리를 실행하는 메서드이고, 인자는 query문이다. query는 쿼리를 조합하는 메서드이며 위 코드에서는 collection(FROM 테이블명), limit(LIMIT)을 합치고 있다. `collection(db, "collectionName")`은 `FROM collectionName`과 같은 문장이라고 볼 수 있다. getDocs에서의 get이 `SELECT *`이라고 보면 되고, limit 메서드는 `LIMIT`이다. 따라서 전체 문장은 `SELECT * FROM collectionName LIMIT 1`과 동일한 문장인 셈이다.

- SQL vs NoSQL 용어 비교
  | 구분 | SQL (Relational DB) | NoSQL (Document DB) | 비고 |
  | :--- | :--- | :--- | :--- |
  | 저장소 | Table (테이블) | Collection (콜렉션) | 데이터들의 집합체 |
  | 개별 데이터 | Row / Record (행) | Document (다큐먼트) | 실제 정보가 담긴 하나의 단위 (JSON 등) |
  | 데이터 항목 | Column (열) | Field (필드) | 속성 이름 |
