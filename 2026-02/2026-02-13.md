# 2026-02-13 TIL

## 오늘 배운 것

- FSD(Feature-Sliced Design)

- FSD는 총 7개의 레이어로 구성되며, 위에서 아래로만 참조할 수 있다는 엄격한 규칙을 가진다. 최근에는 Processes 레이어를 잘 사용하지 않아 보통 6개의 레이어로만 구성한다.
  - App: Provider, 앱 설정 등 전역적인 설정이나 데이터.
  - Processes(선택사항): 여러 페이지에 걸친 복잡한 비즈니스 로직.
  - Pages: 앱의 각 화면 단위.
  - Widgets: 완성된 UI 블록. (예: Header, ProductList)
  - Features: 사용자가 수행하는 기능 단위. (예: AddToCart, SearchProducts)
  - Entities: 비즈니스 실체. (예: User, Order, Product)
  - Shared: 특정 비즈니스에 종속되지 않은 재사용 가능 요소. (예: Button, Input, Utils)
- 모든 레이어 내부의 구조는 세 단계로 나뉜다.
  - Layer: 위에서 설명한 7단위. (예: features/)
  - Slice: 해당 레이어 내의 구체적인 주제. (예: features/auth/)
  - Segment: 코드의 역할별 분류. (예: ui, model, api, lib)
- 각 슬라이스는 외부로 노출할 것만 index.ts에 정의한다. 이를 통해 슬라이스 내부의 구현 세부 사항을 숨기고(캡슐화), 다른 모듈과의 의존성을 관리할 수 있다.
