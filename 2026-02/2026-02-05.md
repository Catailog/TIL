## 오늘 배운 것

- Firebase Authentication

- 과거에는 인라인 요소가 블록 요소를 감쌀 수 없었으나, HTML5부터는 a 태그가 예외적으로 div, h1 같은 블록 요소들을 감싸는 것이 문법적으로 허용된다. a 태그를 기반으로 만들어진 Link 컴포넌트 또한 그렇게 할 수 있지만 스크린 리더의 모호함, 시맨틱 마크업 미준수, 텍스트 드래그 불가 등의 이유로 권장되지 않는다. 이러한 문제점들을 해결하기 위해선 가상 요소를 활용하는 것이 좋다.

```tsx
<article className="relative">
  <h3>
    <Link to="/" className="after:absolute after:inset-0">
      제목
    </Link>
  </h3>
  <p>내용</p>
  <button className="relative z-10">버튼</button>
</article>
```

- 시맨틱 태그 중에서 페이지에 1회만 등장해야 한다고 명시적으로 규정된 태그는 main이 유일하다. header, nav, footer 등의 태그는 article, section과 같은 태그 내에서 각각 등장할 수 있다.

- React Router 사용 시 동적 라우트는 범용적인 매칭을 수행하기 때문에, 더 구체적인 경로를 아래에 두면 의도치 않게 경로를 가로챌 위험이 있다. v5까지는 무조건 위에서부터 차례로, React Router v6부터는 알고리즘이 개선되어 구체성이 높은 경로를 우선적으로 매칭한다.

```tsx
{/* v6 미만에서도 정상 작동하는 구조 */}
<Route path="/posts/write" element={<PostWritePage />} />
<Route path="/posts/:id" element={<PostDetailPage />} />

{/* v6부터는 이 경우에도 정상적으로 작동 */}
<Route path="/posts/:id" element={<PostDetailPage />} />
<Route path="/posts/write" element={<PostWritePage />} />
```
