## 오늘 배운 것

- ref, react-router
- React에서 컴포넌트 태그 사이에 작성된 내용은 자동으로 props.children으로 전달된다. 이 값은 예약된 이름이므로 컴포넌트에서 반드시 children이라는 이름으로 받아야 한다.

```tsx
<Parent>
  <Child1 />
  <Child2 />
  <Child3 />
</Parent>
```

```tsx
function Parent({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>;
}
```

- useState의 값이 업데이트 되면 컴포넌트가 리렌더링되지만, useRef의 값은 업데이트 돼도 컴포넌트가 리렌더링되지 않는다는 차이가 있다. 따라서 렌더링과 직접적인 관계가 없는 값을 저장하거나 DOM에 직접 접근해야 되는 경우 주로 사용한다.
- 컴포넌트에서 ref 속성은 props로 넘겨주지 않는 특수한 속성이다. `<input ref={inputRef} />`라고 쓰면 `inputRef.current`에 DOM 노드나 컴포넌트 인스턴스가 들어간다.

- react-router는 리액트에서 클라이언트 사이드 라우팅을 구현하기 위한 라이브러리이다. 내장 기능이 아니기 때문에 사용하려면 설치해야 한다. BrowserRouter를 앱 전체에 한 번만 적용하면, 이후 모든 링크 이동과 URL 변경에 대해 라우팅이 유지된다. 헤더/푸터를 원하는 링크에만 추가하는 것도 매우 간편하게 할 수 있다.

```tsx
export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        {/* Layout이 있는 페이지 */}
        <Route element={<Layout />}>
          <Route index element={<Home />}></Route>
          <Route path="about" element={<About />}></Route>
          <Route path="my-info" element={<MyInfo />}></Route>
        </Route>
        {/* Layout이 없는 페이지 */}
        <Route path="etc" element={<Etc />}></Route>
        {/* 404 페이지 */}
        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  );
}
```
