# 2026-01-20 TIL

## 오늘 배운 것

- CSS Modules, Styled-Components, clsx
- css 파일을 각 컴포넌트에서 import 해서 사용할 경우 문제점이 있다. 부모 컴포넌트에서 import한 css는 자식 컴포넌트까지 자동으로 적용되며, 클래스 이름이 중복되면 나중에 import된 스타일이 덮어쓰게 될 수 있다. 이를 해결하기 위해 사용하는 것이 module.css이다.
- module.css는 빌드 도구(Webpack, Vite, Next.js 등)에 의해 컴파일 단계에서 js 객체로 바뀐다. 이때 class명에 고유값을 추가해주어 클래스 이름이 겹치지 않는다.

```tsx
export default {
  button: "button__3kF9x",
};
```

- clsx는 React에서 클래스 이름(className)을 조건부로 조합할 때 쓰는 유틸 라이브러리이다. 조건부 className을 깔끔하게 작성할 수 있어 가독성을 높여주고, falsy 값을 자동으로 제거, 객체 문법을 지원해주며 Tailwind의 길어진 className을 상태 단위로 구조화해서 가독성을 높여주기 때문에 Tailwind와의 궁합이 좋다.
- map 메서드 사용 시 key prop에 index를 삽입하지 않도록 주의해야 한다. key를 기준으로 컴포넌트가 재사용 되기 때문에 예기치 못한 결과가 발생할 수 있다.
- key가 app 전체에서 고유값일 필요는 없다. 한 컴포넌트 내에서만 고유하면 된다.
- 이미지 경로를 img 태그의 src 속성에 대입할 때 alias 경로를 인식하지 못 해 파일을 제대로 불러오지 못 한다. 이때 다음과 같이 import를 사용하여 해결할 수 있다.

```tsx
import imgUrl from "@/assets/img.png";
```
