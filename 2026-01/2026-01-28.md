# 2026-01-28 TIL

## 오늘 배운 것

- TanStack Query, Zustand

- TanStack Query는 useState에 비해 다음과 같은 차이점을 갖고 있다.
  - 자동 캐싱
  - 자동 로딩/에러 관리
  - 백그라운드 자동 업데이트
  - 간결한 코드
- useQuery는 주로 GET 요청에, useMutation는 주로 POST, PUT, DELETE 요청에 사용한다. mutate() 호출 → mutationFn 실행 → onSuccess/onError → invalidateQueries → useQuery refetch의 순서로 실행된다.
- Zustand를 쓰면 Context 없이 간편하게 전역으로 상태를 공유하여 사용할 수 있다. Provider Hell 문제를 해결하기 좋다.
