## 오늘 배운 것

- 자바스크립트 json, DOM(접근, 제어, 이벤트)
- appendChild 대신 append를 사용하면 한꺼번에 여러 개의 요소를 추가할 수 있다. 요소뿐만 아니라 텍스트를 추가할 수도 있다.
- 객체를 다루는 것이 아직 익숙하지 않다. 객체에 접근하여 데이터를 가공하거나 보여주는 것은 쉽게 할 수 있지만, 구현 시 어떤 방식을 채택할지는 아직 좀 더 많이 다뤄봐야 자연스럽게 할 수 있을 것 같다. 예를 들어 나는 콜백 내에서 원본 데이터 손상을 방지하기 위해 상수에 스프레드 연산자로 객체를 복사해 가공한 후 그 상수를 반환했다. 하지만 강사님은 반환할 때 리터럴 객체에 스프레드 연산자를 추가하고, 바뀐 엔트리만 추가하여 덮어쓴 후 반환했다. 그 방법이 훨씬 좋아 보였고 그런 구현 방식을 바로 떠올리려면 더 많은 경험이 필요할 것 같다고 느껴졌다.

```js
// 내 코드
const myResults = productsData
  .filter((product) => product.price >= 50000)
  .map((product) => {
    const p = { ...product }; // 원본 데이터를 손상시키지 않기 위함
    p.price = product.price.toLocaleString(); // 여기서 p 대신 원본 데이터를 참조했다면 손상됐을 것
    return p;
  });

// 강사님의 코드
const myResults = productsData
  .filter((product) => product.price >= 50000)
  .map((product) => {
    const price = product.price.toLocaleString();
    return {
      ...product,
      price,
    }; // 새로운 객체를 만들어 반환하기 때문에 원본 데이터가 손상되지 않음
  });
```

- 마우스 이벤트 중 mouseover/mouseout과 mouseenter/mouseleave는 똑같아 보이지만 버블링 여부에 차이가 있다. mouseover/mouseout은 버블링이 발생하여 부모와 자식간의 커서 위치 차이에서도 트리거 되는 반면, mouseenter/mouseleave는 부모와 자식간의 이동은 무시한다.
- 이벤트 전파(propagation)는 요소의 위치가 아니라 DOM의 구조에 따라 이루어진다는 점에 유의해야 한다. 자식 버튼이 부모의 바깥으로 빠져 있어도 자식의 이벤트가 부모에게까지 버블링 된다.
